Text 글자는 타자기 폰트(Courier New 등)로 쓰고, Dungeon 글자는 굵고 거친 돌 질감으로 표현하면 멋질 것

앞서 정의한 시스템(태그, JSON 구조, 카드 카테고리)을 시각적으로 구현할 **프론트엔드 기획서**입니다.
**"이미지 없이(No AI Image) 세련되게"**라는 제약을 장점으로 승화시켜, **모던 다크 판타지(Modern Dark Fantasy) UI**를 지향합니다.

## 반드시 이모지는 사용하지 않고, react-icons만을 사용합니다.

## 예쁜 재질과 문양을 배경에 넣을 수도 있습니다.

# 🎨 UI/UX 디자인 기획서: AI 텍스트 던전 메이커

## 0. 디자인 컨셉 (Design Language)

- **Color Palette:**
  - **Background:** Deep Charcoal (`#121212`) ~ Obsidian (`#000000`)
  - **Text:** Off-White (본문), Silver (보조), Gold (강조/전설), Blood Red (경고/적)
  - **Border/Frame:** 얇은 1px 라인, 네온 글로우보다는 **앤틱한 금속 재질감**의 CSS 그라디언트 사용.
- **Typography:**
  - **Headings:** 명조체 (serif) - 고전적인 판타지 느낌 (예: Cinzel, Noto Serif KR)
  - **UI/Log:** 고딕체 (sans-serif) - 가독성 중심 (예: Pretendard, Roboto)
- **Icons:** 흑백 벡터 아이콘 (RPG Awesome, Game-Icons.net 스타일) 활용.

---

## 1. 🚪 로그인 및 메인 로비 (Entry Page)

_게임의 첫인상. "오래된 지하 감옥 입구" 느낌._

- **Layout:** 중앙 정렬.
- **Visuals:**
  - 배경은 칠흑 같은 어둠 속에서 희미한 먼지가 떠다니는 효과 (CSS Particle).
  - 타이틀 로고는 금속 활자처럼 무겁게 표현.
- **UX Flow:**
  - [Google/Guest 로그인] 버튼.
  - 로그인 성공 시 육중한 철문이 열리는 소리(`Audio`)와 함께 화면이 어두워지며 로비로 진입.

---

## 2. 🏠 메인 허브 (Main Hub / Dashboard)

_플레이어가 다음에 무엇을 할지 결정하는 베이스캠프._

- **Layout:** 3단 그리드 카드 형태.
- **Menu Tabs (GNB):**
  1.  **탐험 (Explore):** 다른 유저의 던전 목록.
  2.  **제작 (Workshop):** 내 던전 관리 및 생성.
  3.  **영웅 (Hero):** 내 캐릭터 스탯 업그레이드 및 도감.
- **Key UI:**
  - **유저 정보:** 상단에 내 닉네임, 보유 골드(Gold), 제작자 포인트(Token) 표시.
  - **알림 센터:** "누군가 당신의 던전에서 사망했습니다 (+50 Gold)" 같은 메시지가 팝업 토스트로 뜸.

---

## 3. 📜 던전 목록 페이지 (Dungeon List)

_길드 의뢰 게시판(Quest Board) 컨셉._

- **List Item (던전 카드):**
  - **제목:** "어둠의 쥐들이 사는 곳" (굵은 명조체)
  - **태그:** `#독(Poison)` `#초보자환영` `#로그라이크` (작은 칩 형태)
  - **난이도:** 해골 아이콘 개수 (💀💀💀)
  - **작성자:** 유저 닉네임 + 평점(⭐4.5)
  - **통계:** 현재 도전 중 5명 | 클리어율 12%
- **Filter/Sort:** 최신순, 인기순, 난이도순, _내가 안 깬 던전만 보기_.

---

## 4. 🛠️ 던전 생성 페이지 (Builder UI)

_가장 중요한 페이지. "마법사의 책상" 컨셉. 화면이 반으로 나뉩니다._

### 4.1 좌측 패널: 창조의 언어 (Chat & Input)

- **UI:** 채팅창 인터페이스.
- **Interaction:**
  - **AI 페르소나:** "던전 마스터"가 상주. *"다음 방에는 무엇을 배치할까요?"*라고 물어봄.
  - **입력창:** 유저가 자연어로 입력 (예: "불을 뿜는 빨간 용을 만들어줘").
  - **기능 버튼:** [몬스터] [함정] [상점] 등 카테고리 숏컷 버튼 제공.

### 4.2 우측 패널: 카드 미리보기 & 덱 (Preview & Deck)

- **Card Preview:**
  - AI가 생성한 JSON 데이터를 즉시 **카드 형태**로 시각화.
  - 상단: 아이콘 (용 그림).
  - 중앙: 이름("화염의 비룡") 및 설명.
  - 하단: 스탯(ATK 50, HP 200) 및 태그(`FIRE`, `DRAGON`).
  - **수정 버튼:** 생성된 카드가 마음에 안 들면 [재생성(Reroll)] 혹은 [수동 수정(Edit)] 클릭.
- **Slot Indicator:**
  - 하단에 `[ ■ ■ ■ □ □ ... ]` 형태로 현재 슬롯 채움 상태 표시 (3/20).
  - 완성된 카드를 드래그하여 순서 변경 가능.

### 4.3 하단 액션 바

- **[던전 게시하기]:** 모든 슬롯이 차면 활성화.

---

## 5. ⚔️ 던전 플레이 페이지 (Gameplay UI)

_텍스트 어드벤처의 현대적 해석. 정보의 가독성이 최우선._

### 5.1 상단: 진행 상황 (Progress)

- `Room 5 / 20` 표시.
- 현재 방의 이름과 분위기 텍스트가 타이핑 효과로 나타남.
  - _"당신은 축축한 지하수로에 들어섰습니다..."_

### 5.2 중앙: 인카운터 (The Encounter)

- **적/오브젝트 카드:** 화면 중앙에 크게 배치.
- **Visual FX:**
  - 공격받을 때: 카드가 붉게 번쩍이며 흔들림 (CSS Animation).
  - 상태이상: 카드 주변에 독 방울이나 불꽃 아이콘이 떠다님.
- **상호작용:** 적 카드를 클릭하여 상태창 확인 가능.

### 5.3 하단: 로그 & 컨트롤 (Log & Control)

- **Log Area (좌측 70%):**
  - 검은 반투명 박스. 텍스트가 아래에서 위로 올라감.
  - 중요 키워드(데미지, 치명타)는 색상 강조 (`<span style="color:red">150 DMG</span>`).
- **Action Deck (우측 30%):**
  - 플레이어의 행동 버튼 4~5개.
  - [공격] [스킬(Q)] [아이템(W)] [도망] [대기].
  - 스킬 버튼 위에 마우스 오버 시 툴팁으로 효과/쿨타임 표시.

### 5.4 사이드/오버레이: 플레이어 상태

- 화면 좌측 하단 구석에 고정.
- **HP/MP Bar:** 직관적인 게이지 형태.
- **Buff/Debuff:** 작은 아이콘 나열. 마우스 오버 시 설명 표시.

---

## 6. 👤 캐릭터 생성/준비 페이지 (Preparation)

_던전 입장 직전, 비장함을 주는 화면._

- **직업 선택:** 타로 카드처럼 나열된 직업 카드 4장(전사, 마법사 등) 중 하나 선택.
  - 선택 시 카드가 뒤집히며 상세 스탯 표시.
- **특전(Perk) 장착:**
  - "가방" 아이콘을 눌러 이번 탐험에 가져갈 유물 1~2개 선택.
  - _예: [오래된 나침반 - 갈림길 힌트 제공]_
- **[입장하기] 버튼:** 누르면 횃불이 켜지는 애니메이션과 함께 로딩.

---

## 7. UI 디테일 & 인터랙션 가이드

### 7.1 로딩 화면 (Loading)

- 검은 화면에 "던전 생성 중..." 텍스트.
- 하단에 랜덤한 **게임 팁(Lore)** 표시.
  - _Tip: 언데드 몬스터는 신성 공격에 취약합니다._
  - _Tip: 미믹은 가끔 진짜 상자보다 깨끗해 보입니다._

### 7.2 반응형 처리 (Mobile vs PC)

- **PC:** 가로형 레이아웃. 로그와 카드가 좌우 혹은 상하로 넓게 배치.
- **Mobile:** 세로형 레이아웃.
  - 상단: 적 카드.
  - 중단: 로그 (3줄만 표시, 터치 시 전체 로그).
  - 하단: 조작 버튼 (엄지손가락 범위).

### 7.3 사운드 (SFX) - 시각 보조

[text](../Documents/docs)

- 이미지가 없으므로 사운드가 중요.
- **버튼 클릭:** 돌 부딪히는 소리, 종이 넘기는 소리.
- **텍스트 출력:** 타자기 소리 혹은 펜 사각거리는 소리 (옵션으로 끄기 가능).
- **전투:** 칼 휘두르는 소리, 타격음, 비명 소리.

이 UI 기획은 **"텍스트를 읽는 피로도"**를 줄이고 **"상황을 직관적으로 파악"**하는 데 초점을 맞췄습니다.

**개발 우선순위:**

1.  **Dungeon Play Page:** 전투 로그가 올라가고 버튼을 누르는 핵심 경험부터 구현.
2.  **Builder UI:** AI와 통신하여 카드가 만들어지는 과정 구현.
3.  **Lobby/List:** 나머지는 데이터를 뿌려주는 껍데기이므로 후순위.

이 UI 구조라면 개발자 혼자서도 CSS 프레임워크(Tailwind CSS 등)를 활용해 충분히 "있어 보이는" 게임을 만들 수 있습니다.
